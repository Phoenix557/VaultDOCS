## Data flow on upload

1. Client sends `multipart/form-data` with `Authorization: Bearer cdn_…` or JWT
2. Server validates magic bytes, size, and user quota
3. File written to `uploads/{discordId}/{uuid}{ext}`
4. Metadata saved to MongoDB
5. Response includes `fullUrl` for tools like ShareX

**Public serving:** `GET /f/{id}` reads from disk, optionally resizes images via Sharp, and returns with cache headers. No session required.
